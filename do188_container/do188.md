
# Chapter 2 Podman Basics #

## Container Networking Basics ##

* 'podman network create -o isolate <network name>' create a network isolated from other networks
* 'podman inspect -f '{{.NetworkSettings.Networks.apps.IPAddress}}'' we can retrieve specific information with the option -f, this method is called json path

# Chapter 3 Container images #

* We can export a container to a TAR file using the command 'container export -o <tar-name> <container-id>' and import them using 'podman import <tar-name> <name:tag>'
* We can save a image to a tar archive using the command 'podman save --output <tar-name> registry.access.redhat.com/ubi8/httpd-24' this example export the image httpd-24 to a tar file
* We can import the image from a tar using the command 'podman load --input <tar-name>'

# Chapter 5 Persisting Data#

* To export a volume we can use the command `podman volume export <volume-name> --output <tarball-name.tar.gz>`
* To import a volume from a tar to a existing volume we can use the command `podman import <existing-voulme> <tarball.tar.gz>`

# Chapter 6 Troubleshooting Containers #

* We can verify the container PID with the command podman inspect <Container-name> --format '{{.State.Pid}}'
* In some cases the container do not have some commands to run any command from container host we can use `nsenter -n -t <container-pid> ss -pant` in this eg we ran the command `ss -pant` into the container

# Chapter 7 Multi-container Applications with Compose #

* A anonymous volume is a volume that you define in the compose file where you only specified the path. For instance:
  services:
  db:
    image: registry.redhat.io/rhel8/postgresql-13
    ...output omitted...
    volumes:
      - /var/lib/postgresql/data
* `podman-compose down -v` remove any anonymous volume that compose create
